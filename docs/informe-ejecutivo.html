Sistema de Gestión Inmobiliaria
Consultora Medina & Asociados

Plataforma web fullstack para la gestión integral de propiedades, clientes, cobranzas y operaciones de alquiler y venta.

Next.js 15 React 19 TypeScript Supabase Tailwind CSS 4 Desplegable en Vercel

01 Resumen Ejecutivo

¿Qué es esta plataforma? Un sistema web completo que digitaliza y automatiza las operaciones de una consultora inmobiliaria: desde el catálogo público de propiedades hasta el panel de gestión privado con cobranzas y recibos.

La solución incluye dos frentes claramente separados: un sitio público donde los visitantes pueden explorar el catálogo de propiedades con galería de fotos y filtros; y un panel administrativo (dashboard) protegido por roles donde el equipo gestiona propiedades, clientes, cobranzas de alquiler y proveedores.

La plataforma está construida con tecnología moderna de alta demanda en el mercado, garantizando escalabilidad, seguridad y facilidad de mantenimiento.

04 Arquitectura de la Aplicación

NAVEGADOR → Cliente React (componentes interactivos, formularios, carruseles)
NEXT.JS → App Router · Server Components · Server Actions · Middleware de sesión
SUPABASE → PostgreSQL · Auth · Storage · Row Level Security (RLS)
VERCEL → Hosting · CDN · CI/CD desde GitHub · Variables de entorno

Patrones de implementación

  • Server Components por defecto: las páginas que leen datos se renderizan en el servidor, sin exponer credenciales al cliente.
  • Client Components solo donde hay interacción: formularios, diálogos, carrusel, navegación.
  • Server Actions para mutaciones (crear, editar, eliminar) con revalidación de caché automática.
  • Middleware de protección que evalúa sesión y rol en cada request antes de dejar pasar al dashboard.

05 Seguridad y Autorización

Roles de usuario

RolAcceso
adminPanel completo: propiedades, clientes, cobranzas, proveedores, usuarios, métricas.
clienteSolo vista de resumen en el dashboard (métricas propias). Sin acceso a módulos de gestión.
públicoSolo catálogo de propiedades activas. Sin acceso al panel.

Flujo de autenticación

  • Registro con verificación por email de confirmación; enlace redirige al callback /auth/callback que establece la sesión.
  • Al confirmar email por primera vez, el usuario ve el modal de bienvenida y queda logueado automáticamente.
  • Recuperación de contraseña con enlace seguro de un uso.
  • Todas las operaciones sensibles del servidor verifican el rol con requireAdmin() antes de ejecutar.
  • Row Level Security (RLS) en PostgreSQL garantiza que incluso con acceso directo a la API solo se lean/escriban los datos permitidos.

07 Módulos Funcionales

🏘️ Catálogo Público

Propiedades con galería de fotos (carrusel táctil), filtros por tipo/operación, modal de detalle con características y descripción.

🏠 Propiedades

Alta, edición y baja lógica. Carga de hasta 10 fotos (5 MB c/u), subida paralela a Storage, vista previa modal.

👤 Clientes

Padrón único de personas (propietarios, inquilinos, ambos). CRUD con validación y baja lógica en cascada.

💰 Cobranzas

Contratos de alquiler con generación automática de todas las cuotas mensuales. Registro de pagos, recibo imprimible.

🔧 Proveedores

Listado de contratistas y servicios vinculados a la consultora. Alta, edición y baja lógica.

📊 Dashboard Ejecutivo

Métricas clave: contratos activos, cobros atrasados, estado de cartera. Vista diferenciada por rol.

👥 Usuarios Admin

Creación y gestión de cuentas del equipo. Asignación de roles, envío de email de activación.

📱 Diseño Mobile-First

Menú hamburguesa, tablas con scroll horizontal, widgets adaptativos. Funciona desde el celular.

Detalle: módulo de Cobranzas

Al crear un contrato de 12 meses, el sistema genera automáticamente las 12 cuotas mensuales en estado Pendiente. El administrador solo hace clic en "Registrar pago" cuando el inquilino abona, completa el monto y la forma de pago, y puede emitir el recibo al instante.

Detalle: galería de imágenes

  • Hasta 10 fotos por propiedad, cada una hasta 5 MB.
  • Subida paralela a Supabase Storage; las URLs quedan en la base de datos ordenadas.
  • Carrusel con gestos táctiles (swipe) en móvil; flechas en desktop; indicadores de posición.
  • Imágenes optimizadas con next/image para carga rápida (lazy loading, formato WebP).

08 Configuración y Despliegue

  • Variables de entorno: URL y clave pública de Supabase; URL del sitio para redirects de auth.
  • Dominio propio: configurable en Vercel y en Supabase (Redirect URLs).
  • CI/CD: cada git push a main desencadena un build y deploy automático en Vercel.
  • Migraciones: el esquema de base de datos se versiona en supabase/migrations/ y se aplica de forma controlada.
  • Límite de upload: configurado a 55 MB en Server Actions para soportar lotes de imágenes.

10 Conclusión

Esta plataforma consolida en un solo producto las tres capas de operación de una inmobiliaria moderna: exposición comercial (sitio público con galería y filtros), gestión operativa (propiedades, clientes, proveedores) y gestión financiera (contratos de alquiler con cuotas automáticas y registro de cobros).

La arquitectura elegida — Next.js + Supabase + Vercel — permite escalar el sistema, agregar nuevos módulos y mantener el código sin fricciones. El diseño mobile-first garantiza que el equipo pueda operar desde cualquier dispositivo.

El sistema es adaptable a cualquier consultora o inmobiliaria con un esfuerzo mínimo de configuración: cambio de branding, variables de entorno y base de datos lista para operar.
